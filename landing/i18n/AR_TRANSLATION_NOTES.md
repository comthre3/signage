# Arabic translation notes — landing

## Sections using Kuwaiti dialect (intentional for brand voice)

- `hero.*` — headline, subhead, CTAs: mixed Kuwaiti + MSA for warmth; headline leans Kuwaiti, CTA buttons use MSA action verbs
- `faq.q*.q` — questions are dialect-inflected for friendliness; answers are MSA for clarity
- `features.pill` ("ليش خنشوف؟") — uses Kuwaiti "ليش" instead of MSA "لماذا"
- `features.sub` — "لما" instead of MSA "عندما" (mild Kuwaiti flavor)

## Sections using MSA only

- nav, features (body), how, pricing, spotlight, faq.q*.a (answers), cta, footer

## Dialect words used (Kuwaiti substitutions for MSA)

- شغّال (kw) → يعمل (msa) — "working / runs" (used in hero.title, meta.title)
- بشلون (kw) → كيف (msa) — "how" (used in faq.q4.q)
- ليش (kw) → لماذا (msa) — "why" (used in features.pill)
- لما (kw) → عندما (msa) — "when" (mild, used in features.sub)
- شوف (kw) → انظر / اكتشف (msa) — "see / look" (used in hero.cta_secondary)
- مو (kw) → ليس (msa) — "not" (used in features.title, features.pay.body)

## Brand name

- Latin "Khanshoof" used in pricing labels (Starter, Growth, Business, Pro) and technical references.
- Arabic "خنشوف" used when fully embedded in Arabic prose (hero, spotlight, footer, features body).

## Notes on tone

- hero.title / hero.title_em: "شاشات شغّالة بدون وجع راس" — Kuwaiti dialect for the punchy headline; a Kuwaiti café owner should smile at this.
- hero.cta_secondary: "شوف كيف يشتغل" — "شوف" (kw) chosen over MSA "اطّلع على طريقة العمل" for brevity and warmth.
- FAQ questions use light dialect to feel conversational; answers stay MSA because that's where the actual product information lives.
- Pricing plan labels (Starter, Growth, Business, Pro, Enterprise) kept in Latin to match the pricing cards on-screen.
