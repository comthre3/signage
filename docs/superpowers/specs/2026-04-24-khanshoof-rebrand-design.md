# Khanshoof Rebrand (Sawwii → Khanshoof) — Design Spec

**Date:** 2026-04-24
**Status:** Ready for implementation planning
**Related:** Plan 2 (player QR UI — feature branch, pre-merge), Plan 3 (admin /pair page — feature branch, pre-merge). This rebrand commits land on the same branch (`feature/player-qr-pairing`) and merge to `main` together as one bundled release.

## Purpose

Rename the product from **Sawwii** (سوّي / "make it") to **Khanshoof** (خنشوف / "khalli nshoof" / "let's see" — and phonetically close to "Artichoke"). Swap the primary domain from `sawwii.com` to `khanshoof.com`. This is a hard cutover; the Arabic-language version and `Khanshoof → خنشوف` wordmark pairing come in a separate sub-project after this lands.

## Scope

**In scope — user-facing text & domain swaps only:**
- Every `Sawwii` string in shipped UI: `landing/`, `frontend/`, `player/`.
- Every `sawwii.com` / `*.sawwii.com` URL in source: `.env`, `frontend/config.js`, `player/config.js`, `landing/` assets, `landing/docker-entrypoint.sh`.
- `<title>`, `<meta og:*>` tags across all three frontends.
- Code comments referencing the brand or domain in `backend/main.py` and `docker-compose.yml`.
- The fallback Postgres connection string default in `backend/db.py` (dead path in production — env always sets DATABASE_URL — but kept consistent for clarity).
- In-flight Plan 2 + 3 docs: `docs/superpowers/plans/2026-04-24-player-qr-pairing.md`, `docs/superpowers/plans/2026-04-24-admin-pair-page.md`, `docs/superpowers/specs/2026-04-24-admin-pair-page-design.md`. These are about to be implemented, so their brand references get updated rather than left as historical records.
- Roadmap memory file: `/home/ahmed/.claude/projects/-home-ahmed-signage/memory/project_signage_saas_roadmap.md` — brand/domain facts updated in place.

**Explicitly out of scope:**
- Postgres DB name, user, and password (`sawwii` / `sawwii` / …). Invisible to users, baked into a running database, no value in renaming; renaming requires dump/restore with downtime.
- Historical plan docs in `docs/superpowers/plans/2026-04-1*–23*` (everything before this session's plans). These document work as it happened; rewriting distorts the log.
- Arabic name (`خنشوف`), language toggle, RTL support, IBM Plex Sans Arabic pairing. All belong to the i18n sub-project that starts after this rebrand merges.
- Logo image file. Text-only `Khanshoof` in IBM Plex Serif ships this round; logo image swap is a follow-up commit when the user uploads the asset.
- Resend email sender domain (`noreply@khanshoof.com`). Emails are still a DEV stub. A note gets added to roadmap memory so whoever wires Resend uses the new domain; no code change here.
- Git commit history rewriting. Old commits continue to mention Sawwii — that's historically accurate.

## Cutover playbook

Designed so user-visible downtime is effectively zero during the switchover and the only irreversible step (Cloudflare route removal) comes last.

**Step 1 — Code (I do):** All text/domain replacements committed to `feature/player-qr-pairing`. Containers NOT yet rebuilt. `sawwii.com` Tunnel routes still live and still work (they serve the old images).

**Step 2 — Cloudflare Tunnel + DNS (user does):** Add four new Public Hostname routes for `app.khanshoof.com`, `play.khanshoof.com`, `api.khanshoof.com`, `lets.khanshoof.com` pointing at the same internal `localhost:3000 / 3001 / 8000 / 3003` targets. Leave the old `*.sawwii.com` routes in place temporarily. Cloudflare-managed CNAMEs at the registrar make the new subdomains resolve.

**Step 3 — DNS sanity check (I do):** `curl -sI https://api.khanshoof.com/health` returns 200. If not, wait for propagation.

**Step 4 — Rebuild (I do):** `docker-compose up -d --build`. Containers pick up the new `.env`. All services now tell browsers to hit `*.khanshoof.com`. Visiting `*.sawwii.com` will still resolve but the SPA will try to fetch `api.khanshoof.com` — that's fine for the changeover window since `api.khanshoof.com` is live.

**Step 5 — Post-cutover smoke (I do):** Full test matrix below. Confirm everything works on khanshoof.com.

**Step 6 — Remove old routes (user does, hard cutover):** In Cloudflare Tunnel → Public Hostnames, delete the four `*.sawwii.com` routes. Optionally delete the `sawwii.com` CNAMEs at the registrar if the user wants the brand fully gone. Past this step, sawwii.com is dark.

**Step 7 — Merge + memory (I do):** Merge `feature/player-qr-pairing` → `main` (bundled: Plans 2 + 3 + rebrand). Update `project_signage_saas_roadmap.md` with the new brand/domain facts, the merge SHA, and promote the Arabic i18n sub-project to "NEXT".

## Rollback

Unlikely given the testing, but safe: the four Cloudflare route removals in step 6 are the only action that isn't `git revert`-able. If we roll back before step 6, rollback is: `git revert <rebrand commits>`, re-run `docker-compose up -d --build`, sawwii.com is back because its Tunnel routes are still live.

Rolling back after step 6 requires the user to re-add `*.sawwii.com` routes manually — annoying but doable.

## Dependencies & assumptions

- User owns `khanshoof.com` and has registrar access.
- Cloudflare Zero Trust Tunnel is already the single ingress for the VPS (existing sawwii setup), so adding khanshoof hostnames is "add four rows and save" — no new tunnel to install.
- The VPS container targets (`localhost:3000 / 3001 / 8000 / 3003`) do not change.
- No ongoing session-dependent state breaks during a container rebuild — backend sessions are JWT-based (stateless), and existing screen_tokens continue to work across rebuilds.
- The Arabic sub-project is not started. No language toggle UI yet, no translation catalog, no RTL CSS work.

## Non-functional requirements

- **No visual regression** beyond the name swap. The pastel theme, IBM Plex fonts, layout, responsive breakpoints — all preserved pixel-for-pixel.
- **No DB migration.** Zero schema changes. Zero data changes. Existing orgs / users / screens / media survive untouched.
- **No broken deep links** on khanshoof.com. Every prior Sawwii URL structure is replicated on khanshoof.com (`/pair`, `/#signup`, `/screens/*`, etc.).
