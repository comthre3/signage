Khanshoof API

v1 — agent-friendly REST API for managing media, playlists, screens, walls, and schedules.

Quick start

Create an API key from the admin dashboard → API Keys tab. Then:

curl https://api.khanshoof.com/playlists \
  -H "Authorization: Bearer khan_live_..."

Authentication

Every request must include an Authorization header:

Authorization: Bearer khan_live_<your-key>

Keys are tied to your organization. They start with khan_live_ followed by ~32 random characters. Treat them like passwords — never commit them to source control or share them in client-side code.

Scopes

ScopeAllowed methods
api:readGET — list/read media, playlists, screens, walls, schedules
api:rwEverything in api:read plus POST/PUT/PATCH/DELETE on those resources

Each API key is bound to one scope at creation time. Rotate to upgrade — there's no in-place scope change.

Endpoint reference

Media

  • GET /media — list your media library
  • GET /media/{id} — get one media item
  • POST /media/upload — multipart upload (returns id + url)
  • POST /media/url — add a URL-based media item
  • DELETE /media/{id} — delete a media item

Playlists

  • GET /playlists · GET /playlists/{id}
  • POST /playlists — create
  • PUT /playlists/{id} · DELETE /playlists/{id}
  • POST /playlists/{id}/items — add item
  • DELETE /playlists/{id}/items/{item_id} — remove item

Sites + Screens

  • GET /sites
  • GET /screens · GET /screens/{id}/zones
  • PUT /screens/{id} — assign a playlist or schedule

Walls

  • GET /walls · GET /walls/{id}
  • GET /walls/{id}/canvas-playlist
  • POST /walls/{id}/canvas-playlist/items
  • PATCH /walls/{id}/canvas-playlist/items/{item_id}
  • DELETE /walls/{id}/canvas-playlist/items/{item_id}

Schedules (dayparting)

  • GET /schedules · GET /schedules/{id}
  • POST /schedules · PUT /schedules/{id} · DELETE /schedules/{id}
  • PUT /schedules/{id}/rules — replace all rules at once

Organization

  • GET /organization — your org's plan, subscription state, screen limit

What's not in the API (yet)

  • Creating users, sites, or screens (paired with screen-limit logic — coming in a later phase)
  • Triggering billing or signup flows
  • Zone editing (advanced layout)
  • Outbound webhooks
  • MCP server wrapper (coming soon)
